# Medical-Chatbot-with-LLM-Flask (RAG-based)

A **Retrieval-Augmented Generation (RAG)** based medical chatbot that provides **accurate, context-grounded answers** by combining semantic document retrieval with large language model inference over domain-specific medical literature.

---

## Overview

This project implements an end-to-end medical question-answering system that retrieves relevant medical content from PDF documents and uses it to ground responses generated by a large language model, improving factual reliability and reducing hallucinations compared to standalone LLMs.

---

## System Architecture

1. **Document Ingestion**

   * Load medical PDF documents
   * Normalize metadata and retain source information

2. **Text Chunking & Embeddings**

   * Split documents into semantically coherent chunks
   * Generate dense vector embeddings using Sentence Transformers

3. **Vector Storage & Retrieval**

   * Index embeddings in Pinecone
   * Retrieve top-k most relevant chunks using cosine similarity

4. **Response Generation**

   * Combine retrieved context with user queries using LangChain retrieval chains
   * Generate grounded responses using LLaMA-3.3-70B-Versatile

5. **Application Layer**

   * Flask-based backend and web interface for real-time interaction

---

## Key Features

* End-to-end Retrieval-Augmented Generation (RAG) pipeline
* Semantic search using dense vector embeddings
* Context-grounded medical question answering
* Modular and scalable system design
* Full-stack implementation with Flask

---

## Tech Stack

* **Programming Language:** Python
* **LLM:** LLaMA-3.3-70B-Versatile (Groq)
* **Frameworks:** LangChain, Flask
* **Embeddings:** Sentence Transformers (`all-MiniLM-L6-v2`)
* **Vector Database:** Pinecone

---

## Motivation

Medical question answering requires high factual accuracy. This project mitigates hallucinations in large language models by grounding responses in retrieved medical literature, making the system more reliable for domain-specific use cases.
